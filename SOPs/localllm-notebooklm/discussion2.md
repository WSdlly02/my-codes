下一步建议不要马上生成“漂亮的总索引”，而是先做一层 atom card，它是后面所有章节索引、专题索引、总索引的结构化中间层。

  当前状态是：

  原始资料 -> atom Markdown

  下一步应变成：

  atom Markdown -> atom_cards.jsonl -> 章节索引 / 专题索引 / 课件概览 / 总索引

  1. 先做 Atom Card

  每个 atom 生成一条结构化记录：

  {
    "atom_id": "L001-98fc1640d4",
    "title": "航运代理基本概念",
    "atom_type": "tutorial",
    "chapter": "第一章 国际航运代理概论",
    "priority": "core",
    "one_line": "...",
    "keywords": [],
    "concepts": [],
    "documents": [],
    "processes": [],
    "legal_rules": [],
    "case_points": [],
    "questions": [],
    "related_topics": []
  }

  这一步的作用是把 62 个 Markdown 压缩成稳定索引材料，避免每次生成章节索引都重新读全部正文。

  2. 标注三类 Atom

  用 manifest 先规则标注：

  教程原子 tutorial
  - 第一章/第二章/.../第六章 下的 atom
  - 章节索引主干

  推荐视频原子 recommended_video
  - 资料/王学锋、王雪峰、王雪锋 三件套
  - 专题索引补充材料

  课件概念原子 courseware
  - 资料/*.pdf 单文件
  - 课程总览、概念框架、考试重点校准

  3. 生成章节索引

  只以 tutorial 为主干，生成：

  01_章节索引/
  ├── 第一章_国际航运代理概论.md
  ├── 第二章_航次租船.md
  ...

  每章包含：

  本章一句话
  本章知识地图
  本章 atom 路由表
  核心概念表
  流程/单证/条款表
  高频考题
  易混淆点
  补充推荐视频
  相关课件概念

  4. 生成专题索引

  专题索引不按章节，而按考试检索场景：

  03_专题索引/
  ├── 航运基础.md
  ├── 船舶代理.md
  ├── 货运代理.md
  ├── 班轮运输与提单.md
  ├── 航次租船合同.md
  ├── 定期租船与船舶买卖.md
  ├── 多式联运与集装箱.md
  ├── 货运事故与索赔.md
  ├── 国际海上货运公约.md
  └── 争议解决.md

  专题索引会综合三类 atom：

  主干教程 atom
  + 推荐视频 atom
  + 课件概念 atom

  5. 最后生成总索引

  总索引应该非常短，负责“第一跳定位”：

  00_总索引/
  ├── 00_超短总索引.md
  ├── 01_关键词总索引.md
  ├── 02_问题总索引.md
  ├── 03_单证条款流程索引.md
  └── 04_打印顺序建议.md

  我的建议顺序

  scripts/build_atom_cards.py
  scripts/build_chapter_indexes.py
  scripts/build_topic_indexes.py

  专题索引怎么做

  专题索引不能简单按文件夹生成，因为推荐视频和课件原子的逻辑松散。它应该按“考试问题域”建。

  我建议专题索引来源分三类：

  主干章节索引：提供权威骨架
  推荐视频 atom card：提供补充细节、案例、行业背景
  课件 atom card：提供概念总览、老师压缩重点

  专题索引不是从零读 62 个 atom，而是这样做：

  1. 先人工/规则定义专题清单
  2. 给每个推荐视频/课件 atom 生成 card
  3. 根据 card 的 related_topics 挂载到专题
  4. 生成每个专题索引
